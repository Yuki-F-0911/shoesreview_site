# 情報キュレーションガイドライン

## ⚠️ 重要: 著作権と利用規約について

このディレクトリに含まれていたスクレイピングスクリプトは、**著作権侵害および利用規約違反のリスク**があるため削除されました。

### 削除されたスクリプト

以下のスクリプトは著作権上の問題から削除されました：

1. **ECサイトスクレイピング** (`ec_review_curator.py`)
   - Amazon、楽天、Yahoo!ショッピングからのレビュー収集
   - **問題点**: 各サイトの利用規約でスクレイピングが禁止されている

2. **Web記事スクレイピング** (`crawl.py`)
   - レビュー記事の全文を収集
   - **問題点**: 記事の著作権侵害

3. **価格スクレイピング** (`price_tracker.py`)
   - ECサイトから価格情報を収集
   - **問題点**: 利用規約違反

4. **SNSコンテンツ収集** (`reddit_curator.py`, `x_twitter_curator.py`)
   - Reddit、Twitter/Xからの投稿全文収集
   - **問題点**: 投稿者の著作権、プライバシー問題

5. **ブランドサイトスクレイピング** (`release_tracker.py`)
   - 公式サイトからの情報収集
   - **問題点**: 利用規約違反

## 著作権を遵守した代替方法

### 1. 公式APIの利用

各プラットフォームが提供する公式APIを使用してください：

- **楽天API**: 楽天アフィリエイトパートナー向けの商品情報API
- **Amazon Product Advertising API**: Amazonアソシエイト向けAPI
- **YouTube Data API**: 動画メタデータの取得（コンテンツの転載は禁止）
- **Reddit API**: 利用規約を遵守した範囲での使用

### 2. アフィリエイトリンクの活用

価格比較機能は、スクレイピングではなくアフィリエイトリンクを使用：
- ユーザーを各ECサイトに誘導
- 価格はユーザーが直接各サイトで確認

### 3. ユーザー投稿コンテンツ

オリジナルのレビューコンテンツ：
- サイト独自のレビュー投稿機能を提供
- ユーザーが自分の言葉でレビューを投稿

### 4. 引用の適切な使用

他サイトの情報を参照する場合：
- 必ず出典元を明記
- 引用は必要最小限に留める
- 全文転載は禁止
- 要約・コメントを加える

## 法的リスクの回避

1. **スクレイピングは原則禁止**
   - 各サイトの利用規約で禁止されていることが多い
   - サーバーへの負荷をかける行為は不正アクセスと見なされる可能性

2. **レビュー全文の転載は著作権侵害**
   - レビューは著作物として保護される
   - 引用は適切な範囲で出典を明記

3. **画像の無断使用は禁止**
   - 商品画像は各メーカー・販売者の著作物
   - 使用許可を得るか、公式APIで提供される画像を使用

## 推奨される運用方法

1. **オリジナルコンテンツの作成**
   - 自社でレビューを執筆
   - ユーザー投稿を活用

2. **公式情報源の活用**
   - ブランド公式サイトのプレスリリースを引用
   - 公式APIからの情報取得

3. **適切な引用**
   - 出典を明記
   - 引用部分は最小限に
   - 自分の分析・見解を加える
