## 1. RunRepeat.com（ランリピート）

### 1.1 基本情報
- **URL**: https://runrepeat.com/
- **設立**: 2014年
- **本拠地**: デンマーク
- **サービス開始**: 2014年
- **主要市場**: グローバル（英語圏中心）

### 1.2 サービス概要
RunRepeatは、**世界中のランニングシューズレビューを集約し、科学的な分析と組み合わせたレビューサイト**です。

#### 主要機能
1. **レビュー集約**
   - 世界中の専門家レビューとユーザーレビューを収集
   - 各レビューを0-100点のスコアに数値化
   - Metacriticのシューズ版のようなアプローチ

2. **自社ラボでの科学的検証**
   - シューズを物理的に解剖（半分に切断）
   - クッションの硬度を計測
   - 重量、ドロップ、スタックハイトなどの数値を公開
   - 「RunRepeat Score」という独自の評価システム

3. **詳細なフィルタリング**
   - 用途（ランニング、トレイルランニング等）
   - ブランド
   - 価格帯
   - クッションタイプ
   - ドロップ
   - 幅（ナロー、レギュラー、ワイド）

4. **比較機能**
   - 複数のシューズを並べて比較
   - スコア、価格、特徴を一覧表示

### 1.3 ビジネスモデル

#### 収益源
1. **アフィリエイト収益**
   - 各シューズの詳細ページから販売サイト（Amazon、Nike、Adidas等）へのリンクを設置
   - 購入が発生した際に手数料を得る
   - 主要な収益源と推測

2. **広告収益**
   - サイト内に関連広告を掲載
   - Google AdSense等を使用

3. **データ提供**（推測）
   - 収集・分析したデータをメーカーや小売業者に提供
   - 市場調査データとしての価値

#### コスト構造
- **ラボ検証**: 物理的なシューズ購入・解剖・計測にコストがかかる
- **レビュー収集**: スクレイピングまたはAPI連携による自動収集
- **データ分析**: レビューの数値化・スコアリング

### 1.4 技術スタック（推測）
- **フロントエンド**: React/Vue.js等のモダンなフレームワーク
- **バックエンド**: Node.js/Python等
- **データベース**: PostgreSQL/MongoDB等
- **スクレイピング**: Python（Scrapy、BeautifulSoup等）
- **データ分析**: Python（Pandas、NumPy等）

### 1.5 強み
1. **科学的な検証**: ラボでの物理的な計測データが信頼性を高める
2. **包括的なレビュー集約**: 世界中のレビューを1箇所に集約
3. **数値化**: 0-100点のスコアで比較しやすい
4. **詳細なフィルタリング**: ユーザーのニーズに合わせた検索が可能
5. **ブランド認知**: 10年以上の運営実績でブランドが確立

### 1.6 弱み・課題
1. **コスト**: ラボ検証に物理的なコストがかかる
2. **スケーラビリティ**: すべてのシューズをラボ検証するのは困難
3. **言語**: 主に英語圏向け（日本語対応は限定的）
4. **日本市場**: 日本市場への特化が弱い

### 1.7 日本市場への示唆
- **日本にはRunRepeatの完全な日本版は存在しない**
- 個人のブログやアフィリエイトサイトが乱立している状態
- **「レビュー集約 + 科学的検証」のアプローチは日本でも有効**
- ただし、ラボ検証は高コストのため、MVP段階では「レビュー集約 + AI要約」に特化するのが現実的

## 6. 日本市場の現状

### 6.1 既存サービス
1. **個人ブログ・アフィリエイトサイト**
   - 多数の個人ブログが存在
   - アフィリエイト収益を目的としたサイトが多い
   - 情報が散在している

2. **メーカー公式サイト**
   - 各メーカーが自社製品の情報を提供
   - 客観性に欠ける

3. **ECサイトのレビューセクション**
   - Amazon、楽天等のレビュー
   - 品質にばらつきがある

### 6.2 課題
1. **情報の散在**: レビュー情報が複数のサイトに散らばっている
2. **信頼性**: アフィリエイト目的のサイトが多く、客観性に欠ける
3. **言語の壁**: 海外のレビュー情報にアクセスしにくい
4. **専門性**: 専門的なレビューが少ない

### 6.3 機会
1. **RunRepeatの日本版が存在しない**: 大きな市場機会
2. **AI要約による言語の壁の解消**: 海外のレビューを日本語で提供可能
3. **キュレーションによる価値提供**: 散在する情報を1箇所に集約
4. **実業団選手などの「トップ層」の巻き込み**: 差別化ポイント

## 8. 推奨されるアプローチ（日本市場）

### 8.1 MVP段階
1. **レビュー集約 + AI要約**
   - Web上のレビュー記事を収集・要約
   - YouTube動画レビューを自動要約
   - 複数ソースを統合して1つの要約レビューを生成

2. **日本市場への特化**
   - 日本のランナーが関心のあるブランド（Saucony、Brooks等）を中心に
   - 日本語での情報提供

3. **情報源の明示**

   - 信頼性の確保
